There Are FP Learning Barriers

Sharing FP Has Benefits

  • More programmers writing more functional code: software, libraries, languages
  • Larger community helping FP to become more mainstream, leading to more FP jobs
  • More diversity of ideas, driving innovation

Areas to Consider

Curricula and Pedagogy

  • Set clear learning goals
  • Choose primary emphasis: problem solving, language learning, project completion
  • Consider your target audience; build on what they already know
  • Plan a series of small, achievable tasks leading to the end goals
  • Use a variety of teaching mediums, including visual stimuli

Language Choice

  • The best choice may depend on the individual
  • Syntax has a significant impact on novices
  • Language selection should tie in with learning goals
  • Some possible choices: Lisps (Scheme, Clojure), Haskell, and Elixir

Teaching Tools

  • Professional tools are not built with beginners in mind; keep it simple, and explain tools
  • Beginners are easily frustrated with obscure error messages; choose tools that help students recognise/overcome them
  • Environment set-up can be a major pain point; consider using the cloud

Experience Report 1: NICTA Lambda Ladies Workshop

Overview

  • Free, three-day event held at Red Hat Brisbane in March
  • Content based on the NICTA open source, functional programming in Haskell course
  • First NICTA event to include the Lambda Ladies branding and explicitly invite women
  • Instructors included Tony Morris, Mark Hibberd, and myself, with help from Dave Laing
  • Women were given preference, but men were also able to apply/attend; we accepted 16 women and 11 men

What Went Well

  • On day one, there were 12 women among 23 students (52%); over the next two days, numbers dropped but the percentage remained similar, 53-55% women
  • There was a good ratio of instructors to students
  • Writing built-in Haskell functions enhanced understanding
  • There was lots of fundamental, relevant content
  • The expectation was set at the start that it was OK to not follow every detail

Issues Encountered

  • The value and goals of the course weren't clearly defined
  • Not everyone was familiar with the tools and there was no time set aside for explaining them
  • Several ran into issues with whitespace in Haskell
  • Levels of expertise and pace varied greatly among students
  • Some felt there was too much content

Lessons Learnt

  • If you want women to come to your workshop, invite them
  • Make sure it is clear if your workshop will be a mixed gender event; some prefer a woman-only learning environment
  • Include visual representations of key concepts
  • Offer slides/handouts
  • Make sure all instructors deliver some of the content; different perspectives are valuable
  • Have official answers to exercises readily available

Experience Report 2: Haskell in the Cloud Workshop

Overview

  • Four-hour workshop at Codemania Auckland conference in April
  • Taught by myself and Tony Morris
  • 17 registered participants (all men)
  • Brand new course content, focused on Haskell basics and creating a Scotty web app in the cloud with the OpenShift PaaS

Demo

What Went Well

  • The engagement level was high; people got excited about FP and continued the conversation/learning afterwards
  • Most (if not all) got the app running locally and in the cloud
  • Participants were able to complete some code challenges
  • We set the expectation at the start that we'd have to skip over detail and this set the tone; people commented at the end that it had been a good overview

Issues Encountered

  • There were some network issues that caused unexpected errors; these are hard to predict
  • There was a lot of preliminary content to cover to help students understand the web app; this took longer than expected
  • Some (particularly Windows users) had trouble with their local environments; some had not come with a local database set-up, which meant they could only run the app in the cloud

Lessons Learnt

  • Dealing with environment issues can still be painful in a cloud workshop if people want to work locally; perhaps a cloud IDE set-up and deploy system would be preferable
  • Focusing on producing something (ie: a web app) means less time for learning base concepts; this is a trade-off
  • Trying to cover a number of things in a workshop (Haskell syntax, FP concepts, Haskell web libraries, PaaS) means you sacrifice depth – however, sharing the big picture can be a win
